循迹小车系统README.md 
仓库说明
本仓库为嵌入式组考核循迹小车项目完整代码，包含硬件驱动层、控制算法层、应用逻辑层模块化代码，实现了白底黑线循迹、蓝牙遥控 / 速度调节、超声波 + 舵机避障、OLED 状态显示等功能，系统基于 STM32 主控，PID 控制实现稳定巡线，支持赛道十字路口、连续弯道、锐角弯道等复杂路况。
一、系统结构说明
本系统严格按照硬件驱动层（BSP）、控制算法层（Control）、应用逻辑层（App） 划分模块化结构，各层独立文件，无代码冗余，具体模块及文件划分如下：
1. 硬件驱动层（BSP）
底层外设驱动，负责与硬件直接交互，提供统一驱动接口，供上层调用，文件如下：
Motor.c/.h：电机 GPIO/PWM 初始化、正反转 / 速度控制驱动
LineSensor.c/.h：四路循迹传感器 GPIO 初始化、模拟量 / 数字量采集
OLED.c/.h：OLED 显示屏 IIC/SPI 驱动、字符 / 数字显示接口
USART1.c/.h：蓝牙串口初始化、中断接收、指令解析驱动
SG90.c/.h：舵机 PWM 初始化、角度控制驱动
HC_SR04.c/.h：超声波测距模块 GPIO 初始化、距离采集驱动
Delay.c/.h：微秒 / 毫秒延时驱动，适配外设时序要求
2. 控制算法层（Control）
负责核心控制逻辑实现，基于硬件驱动层接口，提供算法调用接口，文件如下：
LinePosition.c/.h：线偏移量计算、丢线检测算法
PID.c/.h：速度 / 转向 PID 控制算法，包含 P/I/D 参数配置、增量式 / 位置式 PID 计算、积分限幅 / 死区过滤
AvoidObstacle.c/.h：超声波 + 舵机避障算法，实现障碍物检测、方向决策、轨迹回正逻辑
3. 应用逻辑层（App）
系统顶层逻辑，调度硬件驱动层和控制算法层，实现业务功能，文件如下：
main.c：系统主入口，完成外设初始化、多模式（自动循迹 / 蓝牙手动）调度、避障逻辑触发、掉电存储调用、OLED 状态刷新
CarMode.c/.h：小车工作模式（自动 / 手动）切换、速度档位调节逻辑
二、线偏移量计算方法
本小车采用四路循迹传感器（沿小车正前方一字排开，与黑线垂直）采集赛道信息，基于加权平均法计算线偏移量，实现高精度的线位置估计
1. 传感器布局与编号
四路循迹传感器从左到右依次编号为 S1、S2、S3、S4，安装间距与线宽匹配（相邻传感器间距 15mm），传感器中心与小车中轴线对齐。
2. 传感器状态采集
传感器为反射式红外传感器，白底反射光强（输出低电平 / 0），黑线反射光弱（输出高电平 / 1），通过 GPIO 采集各传感器状态，得到 4 位二进制状态值（如 S1=1、S2=1、S3=0、S4=0 → 状态值为 1100）。
3. 偏移量加权计算
为每个传感器分配位置权重，以小车中轴线为原点（偏移量 0），向左为负、向右为正，权重值根据传感器与中轴线的距离设定：
S1（最左）：权重 -3
S2（左中）：权重 -1
S3（右中）：权重 +1
S4（最右）：权重 +3
4. 计算公式
线偏移量 Offset = (S1×(-3) + S2×(-1) + S3×(+1) + S4×(+3)) / (S1+S2+S3+S4)
分母为有效传感器数量（排除丢线时全 0 的情况），保证计算有效性；
偏移量范围：-3 ~ +3，Offset<0 表示小车偏左，Offset>0 表示小车偏右，Offset=0 表示小车正对黑线；
5. 丢线检测
当S1+S2+S3+S4=0（所有传感器均检测到白底），判定为丢线，偏移量置为特殊值 -100，触发丢线恢复策略（直线匀速行驶或按之前轨迹转向，直到重新检测到黑线）。
三、控制策略说明
本小车以PID 闭环控制为核心，结合多模式调度、避障轨迹规划、速度档位调节，实现全流程自主循迹，控制策略分核心循迹控制和辅助功能控制两部分，具体如下：
1. 核心循迹 PID 控制
采用位置式 PID 控制算法，以线偏移量 Offset为输入，左右电机速度差为输出，实现小车转向闭环控制，使小车始终贴合黑线行驶。
（1）PID 控制公式
PID 输出 = Pout + Iout + Dout
比例项 Pout = Offset × P_gain：根据当前偏移量直接调节转向，偏移越大，转向越猛，快速纠正偏差；
积分项 Iout = ∫Offset × I_gain：累积小偏差，消除静态误差，保证小车精准对中，增加积分限幅防止积分饱和；
微分项 Dout = (当前 Offset - 上一帧 Offset) × D_gain × 0.7：根据偏差变化率预判转向，抑制超调和高频抖动，0.7 为微分滤波系数，过滤传感器杂波；
（2）电机速度计算
左轮速度 = 基础速度 + PID 输出右轮速度 = 基础速度 - PID 输出
对计算后的速度做限幅（300~999），防止速度过高 / 过低导致电机堵转或失控；
基础速度可通过蓝牙遥控切换（三档：300/400/500），适配不同赛道路况；
（3）PID 参数调参过程
初始参数：P=116.5，I=1.5，D=1.0 → 问题：转向过猛，连续弯道易冲线，锐角弯道抖动严重；
第一次调参：降低 P 至 85.0 → 改善转向过猛问题，直线行驶稳定，但小偏差纠正较慢；
第二次调参：降低 I 至 0.8，增加积分限幅至 60 → 消除静态误差，避免积分饱和导致的超调；
第三次调参：提高 D 至 2.0，增加微分滤波 → 抑制弯道超调和高频抖动，锐角弯道可平稳通过；
最终参数：P=85.0，I=0.8，D=2.0，积分限幅 60，偏移量死区 2，满足所有赛道路况稳定循迹。
2. 多模式工作控制
支持自动循迹模式和蓝牙手动模式，通过蓝牙指令（Q/E）切换，模式间完全隔离，互不干扰：
自动循迹模式：调度 PID 控制和避障逻辑，小车自主循迹，实时检测障碍物，触发避障策略；
蓝牙手动模式：屏蔽 PID 和避障逻辑，通过蓝牙指令（W/S/A/D）控制小车正反转 / 转向，支持三档速度调节；
3. 障碍物避障控制
采用超声波 + 舵机的主动避障策略，融合固定轨迹规划，避障后可精准回线，不丢失赛道：
1.检测触发：自动循迹时，超声波实时检测前方距离，当距离 < 10cm（阈值）且测距有效，触发避障；
2.紧急停车：小车立即停止，防止碰撞障碍物；
3.方向决策：舵机带动超声波左右扫描（0° 左 / 180° 右），采集左右距离，选择距离更远的一侧为避障方向；
4.轨迹规划：向避障方向固定时间转向→直行离开障碍区→反向相同时间转向（轨迹回正）→直行一小段，让循迹传感器重新抓线；
5.PID 重置：避障完成后，重置 PID 积分项和上一帧偏移量，避免避障后寻迹抖动，实现无缝衔接；
4. 辅助功能控制
OLED 状态显示：实时显示线偏移量、PID 输出、左右电机速度、基础速度、避障次数、循迹时间，方便调试和状态监控；
丢线恢复：检测到丢线后，小车以固定速度直线行驶，重置 PID 参数，直到循迹传感器重新检测到黑线，恢复 PID 循迹。
四、遇到的问题及解决方式
本项目开发过程中遇到了多个硬件驱动、控制逻辑、系统兼容相关的问题，均通过模块化调试、逻辑梳理、参数优化解决，以下为核心问题及详细解决方式：
问题 1：循迹小车丢线后，四路传感器熄灭，小车停止行驶，无法恢复
原因分析
1.丢线处理逻辑中使用continue跳过了后续代码，未持续给电机发送速度指令；
2.电机速度 350 处于电机死区，丢线时速度指令无法驱动电机转动；
3.丢线时跳过了 OLED 刷新代码，导致屏幕熄灭，误以为传感器故障。
解决方式
1.保留continue逻辑，在丢线处理中直接调用Motor_SetSpeed(400,400)，提高速度至电机死区以上，保证电机持续转动；
2.丢线时添加 OLED 显示指令（显示 “LOST LINE”），避免屏幕熄灭，同时保留传感器供电，确保丢线恢复时可正常采集信号；
3.丢线时重置 PID 积分项和上一帧偏移量，避免恢复后 PID 输出突变导致的抖动。
问题 2：超声波避障后，小车偏离赛道，直行跑走，无法重新回到黑线
原因分析
避障采用固定时间盲转，转完后直接直行，未做轨迹回正，车身角度偏离原赛道，循迹传感器无法重新抓线。
解决方式
采用对称轨迹回正策略：
1.记录避障时的转向方向和转向时间；
2.避障直行离开障碍区后，反向相同时间转向，抵消之前的转向角度，使车身回到原赛道方向；
3.回正后再直行一小段，让循迹传感器扫过黑线区域，重新激活 PID 循迹；
4.回正完成后重置 PID 参数，实现无缝衔接，避免抖动。
问题 3：避障时小车 “往有障碍物的方向行驶”，避障逻辑反向
原因分析
电机转向函数（Motor_LeftReversal/Motor_RightReversal）的实际转向方向与逻辑判定的方向相反，且左右电机速度分配与转向需求不匹配。(应该是电机装反了导致的)
解决方式
1.梳理电机转向函数与实际转向的对应关系，调整避障方向决策的分支逻辑，使 “距离更远的一侧” 与实际转向方向一致；
2.优化左右电机速度分配，左转时左轮慢、右轮快，右转时右轮慢、左轮快，保证转向方向与逻辑判定一致。
问题 4：蓝牙模块与自动循迹模式冲突，切换模式后小车行驶异常
原因分析
自动循迹和蓝牙手动模式共用全局速度变量，模式切换时变量未重置，导致速度值被污染，电机接收错误的速度指令。
解决方式
1.为两种模式分配独立的速度变量，自动循迹使用局部速度变量，蓝牙手动使用专用全局速度变量，实现变量隔离；
2.模式切换时添加变量重置和OLED 清屏操作，避免上一模式的参数影响当前模式；
3.蓝牙指令仅在手动模式下生效，自动模式下屏蔽蓝牙速度指令，防止误触发。
问题 5：PID 控制时，小车在连续弯道易冲线，锐角弯道高频抖动
原因分析
1.P 参数过大，导致转向过猛，连续弯道时无法及时跟随黑线，易冲线；
2.I 参数过大，积分饱和导致超调，锐角弯道偏差变化快，积分项累积过快；
3.D 参数过小，无法有效抑制偏差变化率，导致高频抖动，且无微分滤波，传感器杂波会触发 D 项突变。
解决方式
1.降低 P 参数至 85.0，减弱比例项的转向力度，使小车平稳跟随弯道；
2.降低 I 参数至 0.8，设置积分限幅 60，防止积分饱和，仅在有效偏差时累积积分；
3.提高 D 参数至 2.0，添加 0.7 的微分滤波系数，过滤传感器杂波，抑制偏差变化率，减少锐角弯道的抖动；
4.添加偏移量死区（2），过滤微小偏差，避免 PID 频繁调节导致的抖动。
